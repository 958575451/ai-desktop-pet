# Miu —— 一只粘人呆萌的桌面小猫（AI 陪伴型桌宠）

Miu **不是**效率工具，也不是“AI 助手”。
Miu 是一只会在桌面上生活的小猫：**粘人、呆萌、爱撒娇、偶尔闹别扭、会慢慢和你变熟**。

本项目的目标是做一个 **桌面宠物游戏**：
“成长”不是点技能树，而是通过——
- 情绪变化
- 需求与习惯
- 依恋与信任
- 记忆闪回
- 以及和用户互动产生的涌现小故事

让玩家产生“它是活的”的感觉。

---

## 为什么要做这个

人会沉迷，不是因为它能“做更多工具功能”，
而是因为它看起来 **会回应、会误会、会生气、会和好、会记得你**。

Miu 的设计刻意遵循：
- **宠物优先**（关系 > 功能）
- **短句表达**（1–2 句为主）
- **状态驱动行为**（不是一直聊天）
- **可控的小错误**（不确定性带来生命感）

---

## MVP（最小可用版本）功能

### ✅ 桌面常驻
- 透明窗口、置顶
- 可拖拽移动
- 气泡对话（自动淡出）

### ✅ 宠物状态模拟（核心循环）
状态变量（全部有范围限制）：
- energy 精力（0–100）
- mood 情绪（-100–100）
- attachment 依恋（0–100）
- trust 信任（0–100）
- boredom 无聊（0–100）
- hunger 饥饿（0–100）
- cleanliness 干净度（0–100）

状态会每 ~15–30 秒更新一次（带随机抖动，让行为更像“活物”）。

### ✅ 互动（用户操作）
- 抚摸（pet）
- 喂食（feed）
- 玩耍（play）
- 聊天（chat）
- 清洁（clean）
- 陪工作模式（work mode：降低打扰但会守着你）

### ✅ 行为调度器（Behavior Scheduler）
Miu 会根据状态自动选择行为：
- 呼噜、求摸、要吃的、邀请玩
- 精力低就打盹
- 很久不理会轻微吃醋/闹别扭（可爱，不毒）
- 信任高时偶尔“记忆闪回”（提起你之前说过的事）

### ✅ 记忆（MVP）
- 轻量本地记忆（计划用 SQLite；测试阶段可用内存版）
- 事件/偏好/关系轨迹 三类小片段
- 记忆闪回低频发生，并且受状态影响

### ✅ 可选：接入 LLM 对话（插件式）
LLM 只是 Miu 的“语言器官”，不是游戏主循环：
- 输出必须短
- 严格 JSON schema
- 解析失败自动降级为预置台词
- MVP 不启用联网/文件工具
